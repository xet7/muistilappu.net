3. Tietokanta. Tämän osaamista kysyi tulevan kesätyön järjestäjä.

  1. Tee uusi sivu, klikkaa New, ja nimeksi tyoharjoittelu/tietokanta.html
  2. Lisää sivulle otsikko Tietokanta, sekä title että h1 align="center" tageihin, kuten aiemmilla sivuilla
  3. Lisää linkit Wikipediaan suomeksi (jos ei löydy, englanniksi), mitä tarkoittaa SQL tietokanta, noSQL tietokanta
  4. Lataa SQLite (komentorivi ohjelma exe) ja DBGate (GUI ohjelma). Lisää tietokanta.html sivulle linkit mistä ne löytyy.
  5. Lataa MSSQL (Microsoft SQL Server) uusin virallinen kokeiluversio, jos mahdollista. Jos ei löydy, kirjoita se tietokanta.html sivulle.
  6. Tee uusi SQLite tietokanta, nimelle muistilappu.db . Siihen lisää taulut: users, boards, swimlanes, lists, cards . Tauluissa voi olla kenttiä kuten name, title.
  7. Kirjoita tietokanta.html sivulle SQL koodi, joka tekee taulut, esim. CREATE TABLE
  8. Kirjoita tietokanta.html sivulle SQL koodi, joka lisää käyttäjän, esim. INSERT INTO users
  9. Kirjoita tietokanta.html sivulle SQL koodi, joka muuttaa käyttäjää, esim. UPDATE users SET name=
  10. Kirjoita tietokanta.html sivulle SQL koodi, joka poistaa käyttäjän, esim. DELETE FROM users WHERE name=
  11. Testaa että ne SQL komennot toimii.
  12. Lisää muistilappu.db tietokanta kansioon tyoharjoittelu.
  13. Lisää tietokanta.html sivulle linkki wikipediaan, ohjesivuihin, tai videoihin, mikä on MSSQL (Microsoft SQL Server) tietokannassa triggeri (trigger)
  14. Lisää tietokanta.html sivulle esimerkki koodi triggeristä.
  15. Asenna PostgreSQL tietokanta. Lisää sivulle linkki wikipedia sivulle, PostgreSQL sivulle, lataussivulle.
  16. Kokeile lisätä taulu PostgreSQL tietokantaan DBGate ohjelmalla jonka latasit kohdassa 3.
  17. Etsi Google tai YouTube haulla miten tehdään PostgreSQL triggeri (trigger). Kirjoita tietokanta.html sivulle linkki triggeri ohjeisiin, ja esimerkki triggeristä.
  18. Etsi tietokanta linkki sivuja, lisää ne tietokanta.html sivulle, esim: Awesome SQLite, Awesome PostgreSQL, Awesome MSSQL
  19. Etsi Google tai YouTube haulla miten MSSQL triggerin voisi muuttaa PostgreSQL triggeriksi. Tämä on siis mistä kesätyössä on varsinaisesti kyse. Voiko triggerit muuttaa kaikki kerralla? Tai onko jossain PostgreSQL lisäosa jolla toimii myös MSSQL triggerit? Miten siirtyä kalliimmasta MSSQL tietokannasta halvempaan PostgreSQL tietokantaan? Mitä vaiheita siirtymisessä on? Miten muuttaa kaikki SQL kyselyt, paljon triggereitä jne kaikki kerralla? Kirjoita tietokanta.html sivulle mitä tietoja löysit, linkit nettisivuihin ja videoihin, onnistuiko vai eikö.
  20. Voiko käyttää jotain tietokanta abstraktiota (DAL, ORM) jolla toimii kaikki tietokannat? Kokeile jotain koodiesimerkkiä käyttää MSSQL, PostgreSQL, SQLite tietokantaa näillä: https://www.npmjs.com/package/typeorm ja asenna myös Node.js osoitteesta https://nodejs.org ja http://www.web2py.com jossa DAL. Kirjoita tietokanta.html sivulle saitko toimimaan ja millä tavalla, koodi esimerkit. 
    git clone https://github.com/web2py/web2py

cd web2py
                
sudo apt -y install python3-tk sqlite3 python3-postgresql python3-pymssql python3-cx-oracle python3-pyodbc python3-pymssql python3-pymongo python3-gridfs python3-imaplib2                    

python3 web2py.py
    Kirjoita salasana ja klikkaa Start. http://127.0.0.1/admin Sinne kirjoita salasana.

  4. Tee ssh avain komennolla ssh-keygen ja painele enteriä. Kopioi ~/.ssh/id_rsa.pub GitHub asetuksiin SSH avaimiin.

  9. Kirjaudu sivulle http://localhost:4000/sign-in
